# 🌐 TIA Protocol: FEC Network Tuning Guide
คู่มือการปรับจูนพารามิเตอร์เครือข่ายสำหรับสถาปัตยกรรม UKB Protocol (Stateless Validation Simulation)

## 🎛️ คำอธิบายตัวแปรในระบบเครือข่าย (Network Parameters)
โปรเจกต์นี้จำลองโปรโตคอลการสื่อสารแบบ Stateless (ไม่รอการตอบกลับ) เพื่อหลีกเลี่ยงปัญหาความล่าช้า (Latency) จาก Handshake ระบบเปิดให้ตั้งค่าพารามิเตอร์ 4 ตัว ดังนี้:

1. **Loss Rate (The Void Loss %):** - อัตรา Packet Loss ภายในช่องว่างมิติ ยิ่งสูงยิ่งอันตราย (ค่า Default จำลองไว้ที่สภาวะโหดร้าย 80-90%)
2. **Base Burst Size (Pods):** - จำนวน Knowledge Pods ที่ถูกหั่นแบ่งตามมาตรฐาน ยิ่งหั่นเยอะ โอกาสที่ข้อมูลรอดจะยิ่งเฉลี่ยกันไป
3. **AI Threshold (Minimum Recovery %):** - ความสามารถขั้นต่ำที่ AI ปลายทางต้องการ (เช่น 20% แปลว่าจากข้อมูล 50 ชิ้น AI ต้องการแค่ 10 ชิ้น ก็สามารถเดาบริบทประกอบร่างคำศัพท์ได้แล้ว)
4. **🛡️ FEC Redundancy (%): [พารามิเตอร์แก้อาการเครือข่ายล่ม]**
   - **Forward Error Correction (FEC)** เป็นเทคนิคมาตรฐานในวิศวกรรมเครือข่าย (ใช้บ่อยใน UDP/Streaming) 
   - คือการส่ง **"ข้อมูลสำรอง (Parity Data)"** แนบเพิ่มเข้าไปใน Burst ชุดเดิม หากตั้งค่า 20% และ Base Burst คือ 50 ระบบจะส่งเม็ดข้อมูลพิเศษไปเพิ่มอีก 10 ชิ้น (รวมเป็นส่ง 60 ชิ้น) เพื่อเพิ่มโอกาสให้ปลายทางได้รับครบตามเป้าหมายโดยไม่ต้องขอส่งใหม่ (Retransmission)

## 🎯 วิธีปรับจูนเพื่อให้ผ่านเกณฑ์ประเมิน (GRA > 80%)

### 1. The Standard Validation (สมดุลทางทฤษฎี)
*ตั้งค่ามาตรฐานเพื่อโชว์การกระจายความเสี่ยง*
* **Loss Rate:** `80%` (โหดร้ายมาตรฐาน)
* **Base Burst:** `50 Pods`
* **AI Threshold:** `20%` (ต้องการ 10 ชิ้น)
* **FEC Redundancy:** `0%` (ไม่ส่งข้อมูลสำรองเลย)
* **ผลลัพธ์ (GRA):** ~ `55% - 65%` ❌ (ระบบพัง! เพราะ 80% loss ทำให้รอดเฉลี่ยแค่ 10 ชิ้น ซึ่งเฉียดฉิวกับที่ AI ต้องการเกินไป)

### 2. The FEC Salvation (การแก้ปัญหาด้วยวิศวกรรมเครือข่าย!) 🛠️
*โชว์ให้อาจารย์เห็นว่า เราแก้ปัญหาระบบล่มในข้อ 1 ได้ยังไงด้วย FEC*
* **Loss Rate:** `80%` 
* **Base Burst:** `50 Pods`
* **AI Threshold:** `20%` (ต้องการ 10 ชิ้นเท่าเดิม)
* **FEC Redundancy:** **`50%`** (เพิ่มข้อมูลสำรองอัดเข้าไปอีก 25 ชิ้น รวมเป็นส่ง 75 ชิ้น)
* **ผลลัพธ์ (GRA):** ~ `90% - 98%` ✅ **(PASS!)**
* **คำอธิบาย:** แม้ Loss จะสูงถึง 80% แต่การที่เราอัด Parity Pods เข้าไปช่วยอีกครึ่งหนึ่ง ทำให้โอกาสที่ฝั่งรับจะได้รับข้อมูลรอดชีวิตรวมกันครบ 10 ชิ้นมีสูงขึ้นมาก ระบบจึงประกอบร่างข้อมูลสำเร็จ!

### 3. The Extreme Void (เมื่อจักรวาลแปรปรวน 90%)
*ดันขีดจำกัดเพื่อรับมือกับ Packet loss ระดับล้างผลาญ 90%*
* **Loss Rate:** `90%` (ข้อมูลหายเกลี้ยง)
* **Base Burst:** `100 Pods` (ซอยข้อมูลละเอียดยิบ)
* **AI Threshold:** `10%` (ให้ AI ใช้สมองขั้นสุด ต้องการแค่ 10 ชิ้น)
* **FEC Redundancy:** `50%` (ส่งสำรองไปช่วยอีก 50 ชิ้น รวมยิง 150 ชิ้น)
* **ผลลัพธ์ (GRA):** ~ `92% - 95%` ✅ **(PASS!)**

**🎓 บทสรุปทางวิศวกรรมเครือข่าย:** Simulator ตัวนี้พิสูจน์ให้เห็นว่า เมื่อเราไม่สามารถใช้ TCP (ที่ต้องมี ACK/Retransmission) ได้ในสภาพแวดล้อมมิติ The Void การแก้ปัญหาที่ถูกต้องตามหลักวิศวกรรมคือการประยุกต์ใช้ **Forward Error Correction (FEC)** ร่วมกับ **Generative AI** เพื่อเติมเต็มข้อมูลที่แหว่งหายไปครับ
